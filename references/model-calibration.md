# 模型校准说明

最后拟合时间：2026-05-19。

## 当前 Elo + 最近状态拟合

数据集：

- 来源：`data/processed/match-features.csv`
- 比赛：已有比分的国际比赛
- 日期过滤：从 1990-01-01 开始
- 训练集：1990-01-01 到 2021-12-31
- 验证集：2022-01-01 到 2026-03-31
- 训练行数：27,719
- 验证行数：4,421

模型：

```text
home_strength = eloCoef * (eloDiff / 100) + formCoef * recentFormDiff + homeCoef * homeAdvantage
away_strength = -home_strength
draw_logit = drawBias
  - drawEloPenalty * abs(eloDiff / 100)
  - drawFormPenalty * abs(recentFormDiff)
  + drawNeutralBoost * neutralFlag

probabilities = softmax(home_strength, draw_logit, away_strength)
```

- 非中立场比赛中，主场优势仍然有意义。
- 验证集中平局概率略微低估。

- 验证集：2022-01-01 到 2026-03-31

| 验证 | 0.879709 | 0.517718 | 0.598507 |

与 Elo + 状态模型的验证 Log Loss `0.879746` 相比，FIFA 只改善了 `0.000037`。拟合出的 `fifaCoef` 接近 0，所以当 Elo 可用时，FIFA 积分应保持为低权重辅助特征。

- 如果 Elo 缺失，FIFA 积分可以获得更高降级权重。

```text
赔率 + 阵容评分 / 身价 + 球队特定平局/主客倾向
```

## 2026-06-15 事后诊断：平局预测系统性低估

2026 世界杯开赛后，实际赛果中平局比例高于模型预测，触发全链路诊断。

### 实际观测数据（截至 4 场完赛）

| 场次 | 模型预测方向 | 模型平局概率 | 实际赛果 |
|:----|:-----------|:----------:|:--------:|
| 卡塔尔 vs 瑞士 | 客胜(78.9%) high | 13.8% | 1-1 平局 |
| 巴西 vs 摩洛哥 | 主胜(51.6%) high | 23.1% | 1-1 平局 |

全 72 场平局概率范围：**8.6% ~ 27.4%**，均值 **20.1%**。没有一场超过 28%。

### 根因分析：4 层叠加压缩

```
校准层 drawBias=-0.15 ──→ softmax 压缩 draw logit
      ↓
外部特征融合(50/50 分权) ——→ 将平局进一步拉向市场+校准的加权均值
      ↓
predictedScore 映射阈值 28% 永远达不到
      ↓
最终输出：无平局方向、无平局比分
```

#### 原因 1：drawBias 过负（最深層）

现代稳健版将 drawBias 从倾向模型的 -0.078 调整到 **-0.15**，虽然整体 Log Loss 改善（0.876→0.873），但平局校准退步。加上 `drawEloPenalty=0.045`，两支 Elo 差 100 的队伍：

```
drawLogit = -0.15 - 0.045×1.0 + drawTendency贡献
         ≈ -0.18 through softmax → 平局被严重压缩
```

验证集校准数据显示该问题早已存在：

| 版本 | 验证集实际平局率 | 平均预测平局率 | 偏差 |
|:----|:--------------:|:------------:|:---:|
| Elo+状态 | 22.98% | 22.15% | -0.83pp |
| +倾向 | 22.98% | 21.85% | -1.13pp |
| 现代稳健 | 22.98% | *未单独报告* | *推测更大* |

原始校准文档明确记录："平局概率仍偏低，后续调参应重点关注平局校准"——但该 TODO 未被跟进。

#### 原因 2：外部特征融合不贡献平局

`predict-match.mjs` 的 blend 中，35% 权重给赔率去水后概率、8% 给阵容评分、7% 给身价——**后两者只分配 home/away，不贡献平局**。50% 的基准权重来自校准层（已压低）。赔率去水后的平局概率本身已被 margin 压缩，简单比例法（当前实现）在赔率分布不均时比幂方法 / Shin 法更低。

#### 原因 3：predictedScore 映射阈值过高

```js
if (probs.draw >= 0.28) return "1-1";  // 全72场最高27.4%，永达不到
if (probs.draw >= 0.22 && gap < 0.10) return "1-1";
```

第二个条件在 gap<0.10 时才触发，但模型在平局概率 22-27% 时 homeAwayGap 很少小于 10%，大大降低了触发几率。

#### 原因 4：confidence 定义放大了误导

confidence 仅看 `gap = max_prob - second_max_prob`：

```js
if (gap >= 0.22) return "high";
```

卡塔尔 vs 瑞士 客胜 78.9% vs 平 13.8%，gap=65.1% → high confidence，但方向完全错了。高置信度不保证方向正确。

### 修复优先级

| 优先级 | 改动 | 预期影响 | 风险 |
|:-----:|:----|:--------|:----:|
| P0 | 调 drawBias 从 -0.15 到 -0.08～-0.10 | 平局均值提升 1-3pp | 需重跑验证集校验 Log Loss |
| P0 | predictedScore 阈值从 0.28 降到 0.25 | 平局比分立刻映射出 9 场 | 低，仅影响输出显示 |
| P1 | 赔率去水从简单除法改为幂方法 | 极端赔率场次平局恢复 +1-2pp | 需对比回测验证 |
| P2 | 增加"弱队偷平"情境特征 | 强弱对话平局校准改善 | 高，需特征工程+全量重训练 |
| P2 | confidence 加入"方向一致性"维度 | 高 gap 但方向错误时不标 high | 需定义新标准 |

### 排查速查

当看到预测结果中没有任何平局比分或平局方向时：

1. 检查 `match-predictions-2026.json` 中 draw 最大值：若 < 28%，predictedScore 永不可能为平局
2. 检查 `config/calibrated-model.json` 的 drawBias：-0.15 表示系统性低估
3. 对某场特定比赛，分别打印校准层概率 vs 赔率去水后概率 vs 最终融合概率，定位压缩发生在哪一层
